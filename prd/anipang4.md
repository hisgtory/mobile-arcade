# 애니팡4 (Anipang4) — 기능 기획서

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | Match-3 (타임어택) |
| **레퍼런스** | 애니팡4 (Wemade Play) |
| **핵심 차별점** | 시간 제한 기반 매칭 (크런치3: 이동 제한 / 애니팡4: 시간 제한) |
| **타겟 개발 기간** | 1주 (MVP) |

## 2. 한국 시장 특수성 분석

### 2.1 애니팡 브랜드 파워
- 2012년 카카오 게임 1호 히트작, 한국 모바일 게임 시장 개척
- "국민 게임" 이미지 — 전 연령대 인지도
- 매치-3 장르를 한국 대중화시킨 IP

### 2.2 우리가 배울 점
- **심플한 코어 루프**: 스왑 → 매칭 → 콤보 → 점수. 복잡한 메카닉 없이 재미
- **시간 제한의 긴장감**: 60초 타임어택이 만드는 중독성
- **콤보 피드백**: 연쇄 매칭 시 화려한 이펙트와 점수 폭발
- **소셜 경쟁**: 친구 랭킹이 재방문 유도

### 2.3 우리가 따라할 수 없는 점 & 대안
| 원본 | 우리 대안 |
|------|----------|
| 카카오 플랫폼 소셜 | 로컬 스코어보드 + 공유 기능 |
| 캐릭터 IP (코코몽 등) | 픽셀아트 푸드 타일 (기존 에셋 활용) |
| IAP 아이템 (타임+, 피버) | 스테이지 클리어 보상 + 광고 리워드 |

## 3. 코어 게임 메카닉

### 3.1 기본 규칙
- **8×8 그리드** (표준 매치-3 보드)
- **스왑 매칭**: 인접 타일 스왑으로 3개 이상 같은 타일 매칭
- **시간 제한**: 스테이지별 제한 시간 (60초~40초)
- **목표 점수**: 제한 시간 내 목표 점수 달성

### 3.2 매칭 시스템
- **3매치**: 기본 100점
- **4매치**: 200점 + 라인 폭발 효과
- **5매치 이상**: 500점 + 폭탄 효과
- **콤보**: 연쇄 매칭 시 배수 적용 (x2, x3, ...)

### 3.3 중력 & 리필
- 매칭된 타일 제거 → 위 타일 낙하 → 빈 공간 새 타일 생성
- 연쇄 매칭(캐스케이드) 자동 처리

### 3.4 스테이지 진행
| Stage | 타일 종류 | 시간(초) | 목표 점수 |
|-------|----------|---------|----------|
| 1 | 6 | 60 | 1,500 |
| 2 | 7 | 55 | 3,000 |
| 3 | 7 | 50 | 5,000 |
| 4 | 8 | 45 | 7,000 |
| 5 | 8 | 40 | 10,000 |

## 4. 수익화 전략

### 4.1 한국형 IAP 구조 참고
- 하트 시스템 (생명 = 재도전 비용)
- 보조 아이템 (시간 추가, 폭탄)
- 시즌 패스

### 4.2 MVP 수익화
- 광고 리워드 (실패 시 추가 시간 광고 시청)
- 스테이지 사이 전면 광고

## 5. 기술 구현

### 5.1 아키텍처
```
lib/anipang4/          → Phaser.io 코어 (매칭 로직, 타일, 씬)
web/arcade/src/games/anipang4/ → React UI (HUD, ClearScreen)
```

### 5.2 이벤트 시스템
- `score-update` — { score, combo }
- `time-update` — { timeLeft, maxTime }
- `stage-clear` — { score, timeUsed }
- `game-over` — { score }

### 5.3 타일 에셋 (기존 픽셀아트 활용)
다른 게임(crunch3)과 시각적 차별화를 위해 다른 에셋 조합 사용:
- fruit_watermelon, fruit_grape_red, fruit_cherry, fruit_banana
- boba_taro, boba_strawberry
- cake_chocolate, cake_cheese
- soda_fanta, popsicle_blue

## 6. 결론

애니팡4 클론은 **시간 제한 기반 매치-3**으로, 기존 크런치3(이동 제한)와 명확히 차별화.
한국 시장에서 검증된 "타임어택 매치-3" 포맷을 그대로 활용하되,
카카오 소셜 대신 로컬 스코어/공유 기능으로 대체.
기존 에셋과 인프라를 최대한 재활용하여 1주 내 MVP 출시 목표.
