# 미스터리 타운 - 합치기 & 사건 (Mystery Town - Merge & Cases)

## 1. 개요

**장르**: Merge Puzzle + Mystery Story
**컨셉**: 미스터리 마을에서 단서 아이템을 합성(Merge)하여 사건을 해결하는 퍼즐 게임
**핵심 루프**: 아이템 합성 → 단서 발견 → 사건 해결 → 다음 사건

## 2. 코어 메카닉

### 2.1 머지 보드
- **그리드**: 6×6 보드 (총 36칸)
- **아이템 등장**: 빈 칸에 새 아이템 생성 (탭으로 생성)
- **합성 규칙**: 같은 레벨의 아이템 2개를 드래그하여 합치면 다음 레벨로 진화
- **진화 체인**: Lv1 → Lv2 → Lv3 → Lv4 → Lv5 (단서 아이템)

### 2.2 아이템 시스템
| 레벨 | 테마: 미스터리 타운 | 색상 |
|------|---------------------|------|
| Lv1 | 🔍 돋보기 조각 | #8B5CF6 (보라) |
| Lv2 | 📎 클립 | #3B82F6 (파랑) |
| Lv3 | 📝 메모 | #10B981 (초록) |
| Lv4 | 📋 파일 | #F59E0B (노랑) |
| Lv5 | 🗝️ 열쇠 (단서) | #EF4444 (빨강) |

### 2.3 단서 & 사건 해결
- 각 스테이지는 **목표 단서 수**를 달성하면 클리어
- Lv5 아이템(열쇠)을 만들 때마다 단서 1개 획득
- 스테이지마다 필요한 단서 수가 증가

## 3. 스테이지 설계

| Stage | 목표 단서 | 보드 크기 | 난이도 |
|-------|-----------|-----------|--------|
| 1 | 1개 | 5×5 | 튜토리얼 |
| 2 | 2개 | 5×5 | 쉬움 |
| 3 | 3개 | 6×6 | 보통 |
| 4 | 4개 | 6×6 | 어려움 |
| 5 | 5개 | 6×6 | 매우 어려움 |

### 제약 조건
- **보드 가득 참**: 빈 칸이 0개면 게임 오버
- **합성 불가능**: 같은 레벨 아이템이 인접하지 않으면 합성 불가
- 전략: 보드 공간 관리가 핵심

## 4. 스코어링

- 합성 1회: +10점
- Lv3 생성: +50점 (보너스)
- Lv5(단서) 생성: +200점 (대량 보너스)
- 연속 합성 콤보: ×1.5 보너스

## 5. UI/UX 플로우

```
Title Screen → Stage Select → Playing → Clear/GameOver → Next Stage
```

### 5.1 HUD (상단)
- 현재 스테이지 번호
- 달성한 단서 수 / 목표 단서 수
- 스코어

### 5.2 보드 (중앙)
- 6×6 그리드 (밝은 배경)
- 아이템은 아이콘 + 레벨 표시
- 합성 시 파티클 이펙트
- 빈 칸 탭 → 새 아이템 생성

### 5.3 Clear Screen
- 스코어 요약
- 달성 단서 수
- Next Stage / Retry / Home 버튼

## 6. 머지 엔진 공통화 전략 (merge-core)

### 6.1 현재 구현
- `lib/mystery-town/` 에 mystery-town 전용 머지 로직 구현
- 향후 다른 머지 게임 (#5, #10, #18, #23, #30, #34, #47, #62, #113)은 동일 패턴으로 빠르게 스킨 교체

### 6.2 공통 엔진 추출 타이밍
- **Phase 1**: mystery-town 단독 구현으로 패턴 검증
- **Phase 2**: 두 번째 머지 게임 구현 시 공통 로직 추출 → `lib/merge-core`
- **Phase 3**: 모든 머지 게임이 merge-core 의존

### 6.3 스킨 양산 전략
하나의 머지 엔진 + 테마별 변수:
- 아이템 아이콘/이름
- 배경 색상/테마
- 스토리 텍스트
- 난이도 곡선
